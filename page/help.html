Login Page

The login page manages login using email addresses to identify a user and confirmation numbers to create tickets for the user in the browser.

User Email Address

Your email address serves as your account name, and also as the place to which confirmation numbers will be sent.

If you are making a new account, enter an appropriate email address.

You can have up to 3 email addresses associated with your account. Its a good idea to have at least 2, in case one is not working when you need it.

Browser Ticket

A browser ticket is stored in the browser local memory and enables a user to log in by giving only the email address associated with ticket. There is a separate ticket for each email address and each browser.

A ticket is created using a confirmation number sent to the email address and entered into the browser. A ticket expires after a certain interval: 2 days for the first ticket, 7 days for the second ticket, and 30 days for each ticket thereafter (a new ticket obtained before the previous ticket expires is not counted as a new ticket for these purposes).

Problem Page

At any given time you have a current problem you are working on. The problem page allows you to create problems, select an existing problem, and manage the current problem.

On the problem page:
↓means `show below'
↑means `hide'
Χmeans `mark for deletion'; deletion does not occur until you `Delete Marked Files' or make a new file
+means `remove deletion mark'
⇒means `make a new file with given extension from the file named on the same line'

Problem Names

A problem name can contain only letters, digits, and the special characters dash(-) and underline(_). In addition, it must begin with a letter.

Current Problem Files

The current problem files are the files that you have uploaded or created for the problem. These files are stored in the problem directory and can be viewed, used, or deleted. They are displayed in the `Current Problem File' table, one line per file.

Clicking on a file name will show the file with line numbers to the right, unless the page is too narrow in which case it will show the file below.

Clicking on ↓ will show the file below.

Files that are a single short line cannot be shown but their contents are displayed in { } brackets at the end of the file's line.

Empty files and binary files cannot be shown but their nature is displayed at the end of the file's line.

Displayable files are assumed to be encoded in UTF-8 or PDF. A UTF-8 file that is displayable has its number of lines indicated at the end of the file's line.
